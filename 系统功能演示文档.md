# 自习室预约平台系统功能演示文档

## 一、系统概述

### 1.1 系统简介
自习室预约平台是一套基于 Spring Boot + Vue + uni-app 开发的综合性管理系统，支持 Web 管理端和移动端（微信小程序）双平台使用。

### 1.2 技术架构
- **后端框架**: Spring Boot 2.5+
- **前端框架**: Vue2/Vue3 + Element UI
- **移动端**: uni-app
- **数据库**: MySQL
- **缓存**: Redis
- **权限框架**: Spring Security + JWT

### 1.3 用户角色
1. **超级管理员** (admin): 拥有系统全部权限
2. **管理员** (room_admin): 负责自习室、预约记录等日常管理
3. **普通用户** (student): 学生用户，可使用预约功能

## 四、核心业务流程演示

### 4.1 用户预约流程
1. **用户登录** → 进入首页
2. **点击预约** → 查看自习室列表
3. **选择自习室** → 查看座位分布
4. **选择座位** → 选择预约日期和时间段
5. **提交预约** → 生成预约记录
6. **到馆签到** → 扫码签到（座位状态变为"使用中"）
7. **学习完成** → 签退（座位释放，预约状态变为"已完成"）

**异常情况处理**:
- 迟到 30 分钟未签到 → 系统自动取消预约，计入违规
- 需要暂时离开 → 点击"暂离"（保留座位 30 分钟）
- 暂离超时 → 系统自动释放座位，计入违规

---

### 4.2 违规处理流程
1. **违规行为**:
   - 预约后未签到（爽约）
   - 暂离超时未返回
   - 签到后未签退

2. **违规处理**:
   - 系统自动记录违规次数
   - 累计 3 次违规 → 禁止预约 7 天
   - 累计 5 次违规 → 禁止预约 30 天

3. **违规申诉**:
   - 用户提交申诉（填写理由、上传凭证）
   - 管理员审核申诉
   - 申诉通过 → 清除违规记录

---

### 4.3 权限控制流程
1. **用户登录** → 获取 token 和用户信息
2. **加载菜单** → 根据角色加载对应菜单
3. **访问接口** → 校验权限标识（如 `system:user:list`）
4. **数据权限** → 根据角色数据权限过滤数据
   - 全部数据权限：查看所有数据
   - 本部门数据权限：查看本部门数据
   - 仅本人数据权限：只能查看自己的数据

## 六、常见问题解答

### Q1: 如何实现权限控制？
**A**: 基于 Spring Security + JWT，通过@PreAuthorize 注解控制接口权限，如 `@PreAuthorize("@ss.hasPermi('system:user:list')")`

### Q2: 数据权限如何实现？
**A**: 使用@DataScope 注解，通过 AOP 自动在 SQL 中拼接数据权限过滤条件

### Q3: 签到功能如何实现？
**A**: 生成包含预约信息的二维码，用户扫码后调用签到接口，校验位置和预约信息

### Q4: 暂离功能如何实现？
**A**: 点击暂离后记录暂离时间，30 分钟内返回可继续使用，超时自动释放座位

### Q5: 系统支持多少人并发？
**A**: 使用 Redis 缓存热点数据，支持数百人同时在线预约
